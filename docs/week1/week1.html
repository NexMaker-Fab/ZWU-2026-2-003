网页搭建

✅ 作业清单

  • 收集网页参考示例

🖥️

🌐 搭建网页过程及内容 - 第一部分：GitHub 仓库搭建

📝 第一步：在 GitHub 里创建一个新仓库来托管我们的项目网站

登录 GitHub 账号，点击右上角的 "New repository" 按钮，开始创建一个新的代码仓库，用于托管我们的项目网站。

图1：在 GitHub 中创建新仓库

⚙️ 第二步：取仓库名称，设置公开 / 私有，顺带添加项目说明文件，点击创建仓库

为仓库命名（如 ZWU-2026-2-003），选择仓库可见性（公开或私有），勾选 "Add a README file" 选项添加项目说明文件，然后点击 "Create repository" 按钮完成创建。

图2：配置仓库基本信息

🔧 第三步：清空页面内容，进入仓库设置，左侧选择 Pages

仓库创建完成后，进入仓库的 Settings（设置）页面，在左侧菜单中找到并点击 "Pages" 选项。在 Branch（分支）下拉菜单中选择 "main"，Directory（目录）选择根目录 "root"，然后点击 "Save" 保存设置。页面上方会显示访问链接。

图3：配置 GitHub Pages 设置

⏳ 第四步：等待 GitHub 检索并生成页面

保存设置后，GitHub 会自动检索仓库内容并生成静态网页。等待几分钟，当页面顶部显示绿色对勾和访问链接时，说明网站基础搭建已完成。任何后续的更新和改动都可以实时看到。

图4：GitHub Pages 部署成功

👥 第五步：将小组成员加入 GitHub

在仓库的 Settings → Collaborators 中，通过邮箱邀请团队成员加入仓库。给每位成员的邮箱发送邀请信息，她们接受后即可协同进行项目管理。

图5：邀请团队成员加入仓库

💾 第六步：将仓库克隆到 GitHub Desktop

打开 GitHub Desktop，选择 "Clone a repository from the Internet"，找到刚才创建的仓库，选择本地保存路径，点击 "Clone" 按钮将仓库克隆到本地电脑。

图6：在 GitHub Desktop 中选择仓库

图7：仓库克隆完成

🔨 制作过程记录

📝 第一步：在 GitHub 里创建一个新仓库来托管我们的项目网站

登录 GitHub 账号，点击右上角的 "New repository" 按钮，开始创建一个新的代码仓库，用于托管我们的项目网站。

图1：在 GitHub 中创建新仓库

⚙️ 第二步：取仓库名称，设置公开 / 私有，顺带添加项目说明文件，点击创建仓库

为仓库命名（如 ZWU-2026-2-003），选择仓库可见性（公开或私有），勾选 "Add a README file" 选项添加项目说明文件，然后点击 "Create repository" 按钮完成创建。

图2：配置仓库基本信息

🔧 第三步：清空页面内容，进入仓库设置，左侧选择 Pages

仓库创建完成后，进入仓库的 Settings（设置）页面，在左侧菜单中找到并点击 "Pages" 选项。在 Branch（分支）下拉菜单中选择 "main"，Directory（目录）选择根目录 "root"，然后点击 "Save" 保存设置。页面上方会显示访问链接。

图3：配置 GitHub Pages 设置

⏳ 第四步：等待 GitHub 检索并生成页面

保存设置后，GitHub 会自动检索仓库内容并生成静态网页。等待几分钟，当页面顶部显示绿色对勾和访问链接时，说明网站基础搭建已完成。任何后续的更新和改动都可以实时看到。

图4：GitHub Pages 部署成功

👥 第五步：将小组成员加入 GitHub

在仓库的 Settings → Collaborators 中，通过邮箱邀请团队成员加入仓库。给每位成员的邮箱发送邀请信息，她们接受后即可协同进行项目管理。

图5：邀请团队成员加入仓库

💾 第六步：将仓库克隆到 GitHub Desktop

打开 GitHub Desktop，选择 "Clone a repository from the Internet"，找到刚才创建的仓库，选择本地保存路径，点击 "Clone" 按钮将仓库克隆到本地电脑。

图6：在 GitHub Desktop 中选择仓库

图7：仓库克隆完成
